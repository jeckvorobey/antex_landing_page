# Инструкции для агентов

## Контекст проекта

- Это лендинг AntEx на React 19, TypeScript, Vite и Tailwind CSS.
- Основной UI находится в `src/App.tsx`, глобальные стили и Tailwind-композиции - в `src/styles.css`.
- Точка входа фронтенда: `src/main.tsx`.
- Статические ассеты лежат в `public/`; текущий логотип используется как `/logo.PNG`.
- Небольшой webhook-сервер для заявок находится в `server/lead-webhook.mjs`.
- `page.pen` - дизайн/макет, не меняй его без отдельного запроса.

## Команды

- Установка зависимостей: `npm install`.
- Локальная разработка фронтенда: `npm run dev`.
- Сборка и проверка TypeScript: `npm run build`.
- Preview собранной версии: `npm run preview`.
- Webhook для заявок: `npm run lead:server`.

## Переменные окружения

- Для фронтенда используется `VITE_LEAD_WEBHOOK_URL`.
- Для webhook-сервера нужны `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`; порт задается через `LEAD_WEBHOOK_PORT`, по умолчанию `8787`.
- Не коммить реальные `.env` и токены. Обновляй только примеры, если меняется контракт переменных.

## Правила разработки

- Отвечай пользователю на русском, если он пишет по-русски.
- Сохраняй текущий визуальный язык: темный зелено-золотой лендинг, крупная типографика, карточки с золотыми рамками, акцент на Max/Telegram CTA.
- Не делай широкий редизайн, если пользователь просит точечную правку.
- Тексты лендинга являются пользовательским контентом: меняй формулировки только в рамках задачи.
- Для новых UI-состояний учитывай мобильную версию; в проекте уже есть отдельные мобильные формулировки через `sm:hidden` / `hidden sm:inline`.
- Не добавляй новые зависимости без необходимости. Если зависимость нужна, объясни причину.
- Для ручных правок файлов используй минимальный дифф и сохраняй строгий TypeScript.

## Проверка

- Перед завершением изменений во фронтенде запускай `npm run build`.
- Если менялся `server/lead-webhook.mjs`, дополнительно проверяй синтаксис Node-командой `node --check server/lead-webhook.mjs`.
- Если проверку невозможно выполнить, явно напиши причину и что осталось непроверенным.

## Git

- На момент инициализации в каталоге есть пустой/некорректный `.git`, из-за чего `git status` не работает как в обычном репозитории.
- Не выполняй destructive git-команды и не инициализируй/чинь git без прямого запроса пользователя.
